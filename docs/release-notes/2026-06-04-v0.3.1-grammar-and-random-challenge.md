# v0.3.1 — Grammar questions & randomized Challenge

Date: 2026-06-04

A small follow-up to v0.3. The Challenge (闯关测试) now covers grammar, draws
a fresh random set of questions each attempt, and tries not to repeat the
same questions. Still fully static, no backend, no audio.

## Highlights

- New grammar-style question types: `grammar`, `tense`, `singular_plural`,
  `comparative`.
- Each attempt is 3 questions: it tries to include 1 vocabulary, 1
  comprehension, and 1 grammar-style question (randomly chosen when a group
  has several), and backfills from the other available types when an article
  has no grammar-style question yet.
- Recently shown questions are avoided across attempts.
- Reloading resumes the same attempt; "Try again" gives a new set.
- Light interactivity: step dots, a subtle feedback animation, a small
  completion celebration.

## Question types

`QuestionType` (in `lib/types.ts`) now is:

```ts
type QuestionType =
  | "vocabulary"
  | "detail"
  | "main_idea"
  | "grammar"
  | "tense"
  | "singular_plural"
  | "comparative";
```

The `Question` shape is unchanged, so all v0.3 data stays valid. Articles
do **not** need every type. Kid-friendly names per type:

| Type | 中文 | English |
|---|---|---|
| `vocabulary` | 单词小侦探 | Word Detective |
| `detail` | 细节小达人 | Detail Expert |
| `main_idea` | 主旨小队长 | Main Idea Captain |
| `grammar` | 语法小能手 | Grammar Whiz |
| `tense` | 时态小专家 | Tense Detective |
| `singular_plural` | 单复数小卫士 | Plural Patrol |
| `comparative` | 比较级小高手 | Compare Champ |

## Selection logic (`lib/quiz.ts`)

`selectAttempt(quiz, recentIds, size = 3, rand = Math.random)` is a pure
function:

1. Buckets questions into three groups — **vocabulary**, **comprehension**
   (detail + main_idea), **grammar** (grammar + tense + singular_plural +
   comparative).
2. Draws one per group, in that order, preferring questions whose id is not
   in `recentIds` (falls back to the full group if all were recent).
3. Backfills from the remaining pool until `size` is reached, so an article
   missing a group still yields a full attempt (existing 3-question
   articles are not shortened).
4. Skips malformed questions (correctAnswer must be one of the options).

It is deterministic given `rand`, which keeps it easy to reason about and
test.

## Persistence (`lib/storage.ts`)

- `english_study.quizProgress` now stores `{ questionIds, answers }` per
  article, so a reload restores the **same** attempt rather than
  re-randomizing. Legacy entries without `questionIds` are treated as stale.
- `english_study.quizHistory` stores the last few question ids shown per
  article (capped), feeding the recent-avoidance above.

## UI / interactivity (`components/ArticleQuiz.tsx`)

- Attempt generation happens only in the Start / Try-again handlers (client
  side), never during render or SSR — the server-rendered intro is
  deterministic, so there is no hydration mismatch.
- Step-dot indicator (●○○) shows position and per-question right/wrong.
- A one-shot `quiz-pop` animation on the feedback box and result card,
  disabled under `prefers-reduced-motion`.
- Completion shows 🏆 (perfect) or 🎉 plus the existing encouraging message.
- Accessibility from v0.3 retained (aria-live feedback, aria-disabled
  options with spelled-out state, locale-aware labels).

## Content seeded this round

Grammar-group questions were added to three articles to demonstrate the
feature (others can gain them in a later content round):

- *My First Day at School* — tense, singular/plural, plus an extra detail.
- *The Elephant's Long Nose* — comparative, singular/plural.
- *Why Do We Have Seasons?* — comparative, tense.

## Limitations

- Articles without grammar questions won't show grammar yet; backfill keeps
  the attempt at three so it is not a regression.
- With few questions in a group, recent-avoidance can't always prevent a
  repeat — it degrades gracefully.
- Grammar content is human-authored and should keep being reviewed for
  age-appropriateness and correctness.
- No audio in this round (per scope); a future optional, off-by-default
  toggle could add it.

## Verification

- `npx tsc --noEmit` and `npm run build` (static export) pass.
- `selectAttempt` checked over 2000 runs: always 3 questions, always one per
  group for a full bank, no duplicate ids, correct fallback/backfill, and
  empty input → empty attempt.
- Static HTML for articles renders only the deterministic intro; question
  text/labels appear client-side after Start.
