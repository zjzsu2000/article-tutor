# v0.3 — Challenge UI (闯关测试)

Date: 2026-05-28

This release closes the core learning loop: read → tap to understand →
answer a light quest → know you understood. The per-article quiz **data**
shipped in v0.2; this release adds the **interface** on top of it.

## Highlights

- A kid-friendly Challenge (闯关测试) appears below every article that has
  quiz data.
- Questions are revealed one at a time.
- Each answer gets immediate, encouraging feedback with a hint on wrong
  answers.
- Progress is saved per article in `localStorage`.

---

## How it works

When an article has a `quiz` (all 8 seed articles do), a Challenge card
renders under the reader.

1. **Intro** — a short prompt and a "开始闯关 / Start the challenge" button.
2. **One question at a time** — each question shows a progress chip
   (第 1 / 3 关) and its type name:

   | Type | 中文 | English |
   |---|---|---|
   | `vocabulary` | 单词小侦探 | Word Detective |
   | `detail` | 细节小达人 | Detail Expert |
   | `main_idea` | 主旨小队长 | Main Idea Captain |

3. **Immediate feedback** — tapping an option locks the question:
   - The correct option turns green.
   - A wrong pick turns red.
   - The question's `explanation` is shown as a hint.
4. **Result** — after the last question, a result card shows the score
   (e.g. 闯关完成：2 / 3 关) with an encouraging message and a
   "再闯一次 / Try again" reset.

The tone is light: no leaderboard, no timer, no forced exam styling.

## Progress persistence

Quiz progress is stored client-side under
`english_study.quizProgress`, keyed per article. Each entry records the
chosen option per question. On reopening an article:

- A fully answered quiz restores straight to the result card.
- A partly answered quiz resumes at the first unanswered question.
- An untouched quiz shows the intro.

No backend is involved; the static-export constraint (`output: "export"`)
is unchanged.

## Files

- `components/ArticleQuiz.tsx` — new component (intro / question / result).
- `components/ArticleReader.tsx` — mounts `<ArticleQuiz>` under the article.
